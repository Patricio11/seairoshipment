Metaship Public API

Metaship Support

External-facing API for Metaship platform integrations. Authenticate with clientId/secretKey to obtain a Bearer token, then use it to access bookings, stock exports, sailing schedules, and reference data endpoints.

Authentication

  1. Call POST /public/v2/auth with your clientId and secretKey
  2. Receive a JWT accessToken with a 12-hour expiry
  3. Include the token in all subsequent requests: Authorization: Bearer <token>

Scopes

Your API credentials are assigned scopes that control access:

Scope Description
bookings:read View bookings and container details
bookings:write Create bookings
contacts:read View contacts and transporters
contacts:write Update contact details
shipments:read View products, countries, POI, and consignee profiles
stock:read Read stock export data
stock:write Update pallet weights
sailing-schedules:read Search sailing schedules
tracking:read View container tracking status and events
tracking:write Subscribe containers for tracking
bookings:read View container tracking events (shared with Bookings)
shipment-documents:read List and download final shipment documents

Rate Limiting

The authentication endpoint is rate-limited to 5 requests per minute to prevent brute force attacks.

Upload a document to a booking or order

​

Upload a base64-encoded document and link it to a booking or order. The file is stored in S3 and metadata is persisted for querying. A presigned download URL (valid for 1 hour) is returned in the response.

At least one of bookingId or orderId must be provided.

Requires scope: bookings:write

Body
required
application/json
  • file
    Type: string
    required

    Base64-encoded file content. May include a data URI prefix (e.g. data:application/pdf;base64,...) which will be stripped automatically.

  • mimeType
    Type: string
    required

    MIME type of the file

  • name
    Type: string
    required

    Filename including extension

  • type
    Type: stringenum
    required

    Document classification

    values
    • DOCUMENTATION_PACK
    • PACKING_LIST
    • COMMERCIAL_INVOICE
    • INVOICE
    • SUPPLIER_INVOICE
  • bookingContainerId
    Type: integer nullable

    Optionally links document to a specific container

  • bookingId
    Type: integer

    Links document to a booking. At least one of bookingId or orderId is required.

  • orderId
    Type: integer

    Links document to an order. At least one of bookingId or orderId is required.

Responses
  • application/json

  • application/json
  • application/json

Tracking (Collapsed)

​

Subscribe containers for real-time tracking and retrieve current tracking status with position and event history. Events are merged from multiple sources: carrier APIs (EQUIPMENT/TRANSPORT events following DCSA standards), AIS vessel tracking (zone/terminal enter/exit events), and terminal operating systems. Events are sorted chronologically and include position coordinates where available.

Tracking Operations
  • get/public/v2/tracking/{containerNo}
  • post/public/v2/tracking/subscribe

Shipment Documents

​

Read-only access to final, customer-ready shipment documents (Bill of Lading, Certificate of Origin, Phyto Certificate, Packing List, etc.). Returns signed CloudFront URLs with a 15-minute TTL. Internal-only file types (XLSX templates, cargo dues, SAD500 customs) are excluded.

Shipment Documents Operations
  • get/public/v2/shipments/{shipmentReference}/documents
  • get/public/v2/shipments/{shipmentReference}/documents/{documentId}

List final documents for a shipment

​

Returns the canonical final PDF documents for a shipment. Internal-only file types (XLSX templates, cargo dues, SAD500 customs) are excluded. Each document includes a short-lived signed CloudFront URL (15-minute TTL) the caller can use to download the file directly.

Requires scope: shipment-documents:read
